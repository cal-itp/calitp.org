# DSDL releases

## Publishing a release

The steps to create a release tag are identical to [Benefits](https://docs.calitp.org/benefits/guides/release/). Release candidate tags are skipped.

## Consuming DSDL

A mechanism to consume previously tagged DSDL releases does not exist yet. For now the easiest option is to load the bleeding-edge CSS directly from [`main`](https://github.com/cal-itp/calitp.org/blob/main/src/stylesheets/dsdl/dsdl.css).

```html
<link rel="stylesheet" href="https://www.calitp.org/stylesheets/dsdl/dsdl.css">
```
